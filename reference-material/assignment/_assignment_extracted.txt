Описание проекта 
Отчёт / Файл README должен включать: 

• описание бизнес-задачи

• описание схемы пайплайна

• описание элементов ETL (Extract, Transform, Load)

• описание архитектуры ML-модели

• полученные метрики модели

• визуализации, графики, изображения

AutoML (или кастомная модель) — автоматизированное обучение ML-модели 
Описание используемого метода автоматизации ML-модели. Допускается использование готовой модели AutoML или только автоматизация отдельных элементов архитектуры. 

Описание должно включать в себя:

• описание используемой модели AutoML

• описание автоматизации отдельных элементов пайплайна 
20 баллов

Создание контейнера для пайплайна 
Создание контейнера Docker. В описании проекта необходимо оформить: 

• текст Dockerfile с используемым образом и описанием команд и их функций

• функции контейнеризации в вашем ML-пайплайне (обеспечение безопасности, оптимизация ресурсов и т.д.)

• Список команд git, которые Вы использовали

Мониторинг 
Реализуйте мониторинг обучения модели и работы ML-системы. Описание должно включать в себя: 

• мониторинг качества модели (предотвращение дрейфа данных, контроль качества данных, анализ метрик модели и т.д.)

• мониторинг инфраструктуры (производительность, загрузка CPU, оптимизация ресурсов и т.д.)

Вы можете использовать скриншоты графиков из сервисов (MLFlow, Docker и др.), замеренное время обучения и другие показатели.

Презентация 
Создайте презентацию на 5-7 слайдов с описанием работы проекта. В презентацию должны входить следующие элементы:

Выполнение проекта 
Формат работы: Проект можно выполнить индивидуально, либо работать в группах (2-4 человека). 
Командная работа: Все участники должны в равном объёме распределить задачи между собой и отразить выполненную работу в истории коммитов на отдельных ветках GitHub. Используйте команды git и систему pull request, чтобы можно было отследить прогресс каждого участника. Одинаковую оценку за проект получают все члены команды.
